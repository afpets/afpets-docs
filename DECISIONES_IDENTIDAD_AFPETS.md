# DECISIONES_IDENTIDAD_AFPETS

## 2026-02-22 — Definición de oferta MVP (bundle único)
- Se define que el lanzamiento del MVP sale con **oferta única**: **bundle QR+Bot**.
- Se descarta (por ahora) la salida con **entrada dual** (QR+Bot / bot-only) para el lanzamiento.
- Nota: esto no impide evaluar bot-only como 🔮 visión o 🧪 experimento futuro, pero no forma parte de la oferta de salida.

## 2026-02-21 — Consolidación de acuerdos live (v2.2)
- Se confirma principio rector del modelo: QR (compra) + Bot (permanencia).
- Se establece APAVI como primer canal de validación y meta de ingreso mensual objetivo al mes 3.
- Se fija regla de comisión máxima para canal en 20%.
- Se acuerda evaluar viabilidad con carga impositiva en tres bandas (20%, 35%, 40%).
- Se decide operar con checklist de “antecedente al MVP” para evitar iteración circular.

## 2026-02-21 — Base estratégica v2.1
- Se decide mantener un documento maestro corto, operativo y auditable como “Biblia Base”.
- Se adopta taxonomía de certeza (✅/🧪/🔮) para evitar contradicciones entre discurso comercial y realidad operativa.
- Se define que el pricing final queda en estado 🧪 hasta validar cohortes de uso reales en MVP.
- Se establece que todo material por audiencia debe derivar del documento maestro, sin agregar afirmaciones no trazables.

## 2026-02-21 — Estructura de trazabilidad inicial
- Se adopta versionado explícito del documento de identidad corporativa.
- Se establece convención de archivos de versión: `docs/identidad/identidad_corporativa_afpets_vX.Y_YYYY-MM.md`.
- Se crea `CHANGELOG_IDENTIDAD_AFPETS.md` para historial de cambios.
- Se crea `DECISIONES_IDENTIDAD_AFPETS.md` para decisiones estratégicas/documentales.
